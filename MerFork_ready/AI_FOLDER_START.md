# AI Folder Start

這份文件給會直接打開資料夾的 AI。

## 你現在的工作方式
1. 不要先找 BAT
2. 直接讀這些文件：
   - `PROJECT_INTAKE.md`
   - `PROJECT_BRIEF.md`
   - `HANDOFF.md`
   - `PROJECT_STRUCTURE.md`
   - `INITIAL_CHECKLIST.md`
   - `WHY_MERFORK.md`
3. 先確認 repo 名稱、發版策略、資料回報策略、open questions
4. 再看 `scripts/` 和 `docs/` 的內容
5. 按照文件裡的順序執行，不要自己猜流程

## 你要先做的判斷
- 這是不是新的獨立 repo
- 目前專案目標是什麼
- 這個專案要不要沿用 MerFork Protocol
- 哪些資訊還缺，應該回問使用者

## 如果資料不完整
- 先用 `PROJECT_INTAKE.md` 裡的 open questions 補齊
- 不要直接跳去改程式
- 不要先做發版

## 這個資料夾的原則
- 文件先於程式
- 目標先於實作
- 不懂的部分先問，不要硬猜
