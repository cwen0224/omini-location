# Start Here

這份文件給下一位接手的人或 AI。

## 你現在要做的事
1. 先確認這是不是新專案
2. 讀 [`HANDOFF.md`](./HANDOFF.md)
3. 讀 [`PROJECT_STRUCTURE.md`](./PROJECT_STRUCTURE.md)
4. 依照 `INITIAL_CHECKLIST.md` 建立最小可用版本

## 核心原則
- 不要先做大架構重構
- 先確保能 build、能發版、能接回報
- 每次變更都要有可回溯的版本
- 重要流程要文件化，不要只留在聊天紀錄

## 這個模板預設要支援
- App 版本管理
- 發版流程
- 問題回報
- 遠端設定或資料同步
- 給下一位接手者看的實務紀錄
